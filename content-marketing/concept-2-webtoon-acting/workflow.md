# Concept 2: 웹툰/애니 명장면 실사 연기 (와우 모먼트)

## 목표

본인 얼굴 일관성을 유지하면서 웹툰/애니메이션 명장면을 실사로 재현. **"내가 그 장면에 들어간다면"** 각도로 변형해 저작권 안전선 유지 + 와우 모먼트 극대화.

## 결과물 스펙

| 항목 | 값 |
|------|----|
| 길이 | 15~25초 |
| 비율 | 9:16 |
| 컷 수 | 3~5컷 (원본 1초 인서트 + AI 영상 4컷) |
| 보이스 | 본인 클로닝 보이스 또는 ElevenLabs |
| BGM | 원작 OST 패러디 또는 트렌드 사운드 |

## 저작권 안전선 (필독)

- 원작 그대로 복제 금지 → "변형/패러디" 라인 유지
- 캡션에 **"AI 패러디"** 명시
- 원작 인서트는 1초 이내 + 출처 자막 ("원작: [작품명]")
- 작품 전체 줄거리 노출 금지 (특정 컷만)
- 상업적 광고 영상에 명장면 사용 금지 (광고는 컨셉 1에서 처리)

## 단계별 SOP

### 단계 0. 사전 준비 (영상당 15분)

- 명장면 선정 ([prompts.json](prompts.json)의 `scene_archetypes` 카테고리 참조)
  - 변신 시퀀스, 감정 클로즈업, 액션 전투, 회상 장면 중 픽
- 본인 셀카 준비 (정면 + 45도 + 측면 = 3장)
  - HeyGen Photo Avatar에 등록 (1회만 등록, 재사용)
  - Midjourney `--cref` 시드 URL 생성 (Discord에 셀카 업로드 → URL 복사)

### 단계 1. 장면 분해 (영상당 5분)

원작 명장면을 4요소로 분해:

| 요소 | 예시 (이세계 회귀물 변신씬) |
|------|---------------------------|
| 의상 | 흑색 망토, 은빛 갑옷, 깃털 장식 |
| 배경 | 폐허 신전, 횃불, 안개 |
| 액션 | 뒤로 한 걸음 → 망토 펄럭 → 손에 빛 모임 → 정면 응시 |
| 카메라 | 로우 앵글 → 360도 회전 → 클로즈업 |

prompts.json의 `scene-decomposition-template` 참조.

### 단계 2. 키 프레임 이미지 생성 — Midjourney v7 (영상당 5~10분)

**핵심 파라미터:**
- `--cref [본인 셀카 URL]` — 본인 얼굴 보존
- `--cw 80` — 얼굴 강하게 보존, 의상/스타일은 프롬프트 따라가도록 (cw 100이면 옷도 본인꺼 따라감)
- `--ar 9:16 --v 7 --s 400` — 스타일라이즈 강하게 (시네마틱 효과)

**프롬프트 구조 (영어 권장):**
```
[Scene type: cinematic shot of] + 
[Character description: Korean person + 본인 특징] + 
[Outfit: 의상 묘사] + 
[Background: 배경 묘사] + 
[Mood: 감정/조명] + 
photorealistic film still, shot on Arri Alexa 65mm
--cref [본인 URL] --cw 80 --ar 9:16 --v 7 --s 400
```

**산출:** 4 variants → 가장 본인 얼굴 유지된 것 1장 픽. 의상/배경 부족하면 이미지 reroll.

### 단계 3. 영상화 — Kling 2.5 Pro i2v (영상당 10~15분)

Kling 2.5 Pro는 얼굴 보존 점수가 높음. 자체 ai-studio-saas의 selfie 잡 (10 크레딧) 또는 Kling 직접.

**액션 큐 작성 룰:**
- 영어로 작성 (한국어보다 정확)
- 1초 단위로 동작 분리 ("She raises her hand → light gathers → she releases energy")
- 카메라 무브 명시 ("slow dolly in", "360 orbit", "low angle push")
- Motion brush (Kling Pro 기능): 손/얼굴만 움직이게 마스크

**5초 영상 1개로 부족하면:**
- 키 프레임 2~3개 따로 생성 → 각각 i2v → CapCut에서 컷 연결
- 또는 Kling Multi-Image (시작/끝 프레임 2장 입력) 활용

### 단계 4. 보이스 (영상당 3분)

**옵션 A (강추):** ElevenLabs Voice Clone — 본인 목소리 1분 샘플로 학습된 voice_id 사용
**옵션 B:** ElevenLabs 한국어 기본 보이스 (캐릭터 톤 매칭)
**옵션 C:** 무성 + 원작 OST 인서트 (가장 빠름, 와우 효과 큼)

대사 길이: 영상 20초 기준 30~50 단어 또는 무성.

### 단계 5. 편집 — CapCut (영상당 20~30분)

**컷 구조 표준 (와우 모먼트 극대화):**
```
[0~1초]   훅: 원작 인서트 1컷 + 자막 "이 장면 기억나?"
[1~3초]   AI 영상 컷 1: 의상/배경 변신 모먼트 (느린 모션)
[3~10초]  AI 영상 컷 2: 액션 클라이맥스 (카메라 다이내믹)
[10~17초] AI 영상 컷 3: 감정 클로즈업 + 보이스 라인
[17~20초] CTA: "프로필에서 sora2-video-saas로 만들어보세요"
```

**CapCut 체크리스트:**
- [ ] 원작 인서트 1초 이내 + 출처 자막 ("원작: [작품명]")
- [ ] 비트 매칭 컷 (BGM 비트 ↔ 컷 전환)
- [ ] AI 영상 시작 시 글리치/줌 트랜지션
- [ ] 자막 큰 글씨 + 화면 상단 ("이거 진짜 사람이야?")
- [ ] 마지막 자막: "Made with sora2-video-saas"

### 단계 6. 게시 (영상당 5분)

- 캡션: "POV: 내가 [작품명] 주인공이라면 / AI 패러디"
- 해시태그: hashtag-library.json의 `concept_2_webtoon` 5개 + `core_brand_tags` 3개
- 게시 시각: 금요일 22:00 KST (주말 시청자 노림)
